## 1. 디자인 컨셉 (Design System)
- **타겟층**: 3040 여성을 위한 모바일 최적화 웹 앱
- **톤앤매너**: 피로함을 주지 않는 호텔식 프라이빗 라운지 (Warm-dark / Deep Taupe & Gold)
- **Color Variables**:
  - `--bg-color`: #06110D (에메랄드 블랙)
  - `--surface-color`: #0A1B13 (어두운 그린 - 카드 및 바텀 시트)
  - `--point-color`: #D4AF37 (소프트 매트 골드 - 주요 버튼 텍스트)
  - `--accent-color`: #B59530 (무광 샴페인 골드 - 외곽선 및 테두리)
  - `--text-main`: #F4F7F5 (오프 텍스트 화이트)
  - `--text-sub`: #9CAB9E (포레스트 그레이)

## 2. 주요 페이지 컴포넌트 형태
1. **메인 홈 뷰 (Default View)**:
   - 상단 필터 바: 지역 선택 (아코디언 형태), 마사지 종류, 휴식 공간 형태, 관리사 연령
   - 전체 필터 초기화(🔄) 버튼 추가
   - 다독 초이스 (가로 스와이프 추천 항목)
   - 추천 파트너 리스트 (세로 배열 리스트 렌더링)
2. **필터링 검색 결과 뷰 (Filtered View)**:
   - 선택된 조건들을 AND 매칭하여 맞춤 결과 산출
   - 화면 재배치 및 무한 로테이션 애니메이션 스킵 처리

## 3. 구현된 핵심 기능 (UI 프로토타입)
- **업체 상세페이지 (Profile Bottom Sheet)**:
  - 썸네일 이미지, 가격 표시 단위 "원"으로 수정 적용.
  - 전용 카테고리 태그 메인과 동기화 연동 (`#massage`, `#place`, `#age`).
  - 방문자 리뷰 갯수 및 가상 리뷰 생성 로직 적용.
  - 리뷰 클릭 시 매장 운영자의 답글 등록 기능 추가 (인라인 토글 UI).
  - 예약/채팅 고정 영역 (Bottom Bar) 활성화.
  - 하트 버튼 클릭 시 관심업체(마이페이지 연동 용도) 등록 토글 및 알림.
- **채팅 문의 및 전화 (Chat Bottom Sheet)**:
  - 1:1 채팅 화면 연결, 관심업체 등록 및 통화 버튼 연동.
- **기타 변경 사항**:
  - 푸터(Footer) 영역 추가 (업체 입점 문의 / 관리자 설정 접속)

## 4. 이후 작업 재개 시 참고사항
현재 모든 디자인과 로직은 단일 `index.html` 파일에 들어 있습니다 (Tailwind CSS 기반 + Vanilla JS).
다음에 "진행하던 다독 앱 작업 띄워줘!" 라고 명령하시면, 저(AI)는 시스템 지식 창고와 이 요약본, 그리고 `index.html`을 즉시 스캔하여 우리가 구현해 놓은 맥락(에메랄드 포레스트 디자인, 파트너 호출 로직, 모달 시스템 등)을 전부 기억해 낸 뒤 작업을 이어나가게 됩니다. 

로컬 파일 `index.html`이 완벽하게 저장되어 있으니, 이 파일을 잃어버리지 않게만 동일한 폴더에 잘 유지해 주세요!

## 5. 파이어베이스(Firebase) 연동 및 버그 픽스 현황 (2026-04-09 업데이트)
- **과거 달성 완료**:
  - Firebase Authentication (이메일/비밀번호 방식) 커스텀 ID/PW 연동.
  - 파트너 등록 시 `partners` 컬렉션에 정보 적재 및 회원 정보 `users` 적재 성공.
  - 무한 로딩 버그 수정 완료.

- **최근 완료된 연동 작업 (✅ 완성)**:
  1. 자동 로그인/로그아웃 등 유저 세션 확인(onAuthStateChanged) 및 UI 헤더 동기화 완료
  2. 하드코딩된 더미 파트너 목록 제거 및 Firestore `partners` DB에서 불러오는 동적 출력 성공
  3. 로그인한 유저만 가능한 찜하기(하트) 기능을 Firestore `users` 문서 내 `favorites`에 영구 저장되도록 보강 완료
  4. 파트너 로그인 시 대시보드 내 하드코딩된 데이터들을 실제 Firestore 파트너 데이터(stats 필드)와 동기화 완료

## 6. 다음 진행 가능한 작업 후보 (Next Steps)
이제 주요 골격과 DB 연동 기초가 마무리되었습니다. 실제 앱처럼 유저와 파트너가 상호작용할 수 있도록 다음 기능들을 제안합니다:
1. **실시간 채팅 및 예약 시스템 연동 (Real-time Messaging)**:
   - 가짜 채팅 UI를 넘어, Firestore 실시간 스트림(`onSnapshot`)을 활용하여 유저와 해당 파트너 간 실제 1:1 메시지 통신 구현.
2. **리뷰 작성 및 별점 시스템 (Reviews & Rating)**:
   - 디테일 프로필에서 사용자가 실제 리뷰를 남기면, Firestore `reviews` 컬렉션에 반영되고 파트너의 평점/리뷰 개수가 자동 갱신되도록 연동.
3. **실제 사진 업로드 연동 (Firebase Storage)**:
   - 썸네일과 갤러리 이미지, 프로필 이미지를 더 이상 가짜 사이트(picsum) 링크가 아닌 사용자가 직접 Firebase Storage에 업로드하여 표출.
4. **관리자 어드민 페이지 데이터 연동**:
   - `admin.html` 등에서 승인 대기 중인 파트너들을 보며 활성화(Activate) 시켜주는 승인 절차 개발.
